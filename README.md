# IMDb Sentiment Analyzer

A full-stack Next.js application that accepts a movie name and returns:
- movie metadata (title, poster, year, rating, plot)
- cast list
- audience review snippets
- sentiment score and classification (`positive`, `mixed`, `negative`)
- summary generated by OpenAI when configured, with automatic heuristic fallback

## 1. System Overview

This project is built as a single Next.js app with:
- React UI in `app/`
- server API route in `pages/api/`
- domain logic in `lib/`
- tests in `tests/`

The frontend never calls OMDb/OpenAI directly. It calls your own backend route (`/api/movies`), and the backend handles third-party services securely using environment variables.

## 2. System Flow Architecture

```text
User (Browser)
   |
   v
Next.js Frontend (app/page.tsx + components/SearchBar.tsx)
   |
   | POST /api/movies { movieName }
   v
Next.js API Route (pages/api/movies.ts)
   |-- validate movieName
   |-- fetchMovie(movieName) -> OMDb API
   |-- fetchReviews(imdbId) -> IMDb reviews endpoint
   |-- fetchCast(actors string)
   |-- analyzeSentiment(reviews)
   |-- generateAudienceSummary(...) -> OpenAI (optional)
   |     \-> fallback to heuristic summary if no key/failure
   v
JSON Response (metadata + cast + reviews + insights + summarySource)
   |
   v
Frontend MovieCard render
```

## 3. Detailed Request Lifecycle

1. User enters a movie name in the UI.
2. Frontend sends `POST /api/movies` with `movieName`.
3. API validates input (`>= 2` characters).
4. API calls OMDb by title (`t=<movieName>`, `plot=full`) and gets an IMDb ID.
5. API fetches review snippets using IMDb ID.
6. API computes sentiment:
   - token scoring
   - normalized score
   - class: positive/mixed/negative
   - top highlight terms
7. API tries OpenAI summary (if `OPENAI_API_KEY` exists).
8. On OpenAI failure/unavailable key, API uses heuristic summary.
9. API returns full payload; UI renders results.

## 9. API Contract

Endpoint:
- `POST /api/movies`

Request body:
```json
{
  "movieName": "The Matrix"
}
```

Success response (shape):
```json
{
  "imdbId": "tt0133093",
  "title": "The Matrix",
  "poster": "https://...",
  "year": "1999",
  "rating": "8.7",
  "plot": "Full plot...",
  "cast": ["Keanu Reeves", "Laurence Fishburne"],
  "reviews": {
    "count": 12,
    "samples": ["...", "..."]
  },
  "insights": {
    "score": 1.75,
    "classification": "positive",
    "audienceSummary": "...",
    "highlights": ["performances", "writing"]
  },
  "summarySource": "openai"
}
```

Common errors:
- `400`: invalid/missing movie name
- `405`: wrong HTTP method
- `500`: upstream failure/internal processing issue

## 13. Troubleshooting

- `Failed to fetch movie details`:
  - check `OMDB_API_KEY` in `.env.local`/Vercel env
  - ensure API quota/key validity in OMDb account

- Always getting heuristic summary:
  - check `OPENAI_API_KEY`
  - inspect Vercel/function logs for OpenAI call errors

- Empty/limited review samples:
  - IMDb review HTML can vary; app falls back to context-derived reviews

- Build fails on Vercel:
  - run `npm run build` locally first
  - confirm env vars exist in production environment

## 14. Security Notes

- Keep API keys server-side only.
- Do not expose keys in frontend code or commit `.env.local`.
- Rotate keys if they are leaked.
